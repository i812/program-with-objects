Tags and Elements

Tags are always contained within angled brackets

An opening tag is the left angled bracket, followed by the name of the tag, then the right angled bracket. A closing tag is the left angled bracket, followed by a slash ("/") and "the name of the tag, then the right angled bracket.

Most tags are structured with an opening tag and a closing tag. Some tags do not require a closing tag. These are called "Void" tags.

A tag as a whole is called an element. An HTML element refers to everything within a set of opening and closing tags.

This video explains the distinction between tags & elements well.

Attributes

An Attribute is the property assigned to an HTML element (eg. the href attribute of an anchor tag)

Inline Elements and Block Elements

Inline elements are just text.

Lesson 3: Adding CSS to HTML Structure

CSS - Cascading Style Sheet

- Code written by programmers to control the style of related HTML elements, by giving similar HTML elements the same class name and then specifying the style that should apply to that given class.

- "Cascading means that rules are applied not only to the elements they directly match, but those elements' child elements. However, if a child element has multiple, overlapping rules defined for it, the more specific rule takes place.

Avoids Repetition

Repeatedly doing the same thing over & over in different HTML elements to give them a specific style results in repeated code or very similar code with minor differences. When changes need to be made, it is difficult to remember every place that style is repeated in the HTML elements. Having to manually make those changes in every location can lead to typos or errors, or result in a difference in style where the programmer might want the same style applied.

Consistency

Specify a style for a given class in one place (CSS style sheet), and assign that class to multiple parts of the HTML document. This makes it easy to make changes to the styles, as you only have to make the change to the class in CSS, resulting in the same change to every part of the HTML document that class is assigned to.

Inheritance

The mechanism by which properties are applied not only to a specified element, but also to its child elements.

Relies on the document tree, the hierarchy of HTML elements in a page based on nesting.

It matters where you define the rules and in what order they are applied. Styles can be defined in different places. They are applied in the following order, with definitions further down the list overwriting previous definitions:

1. default stylesheet of a browser (different browsers have slightly different styles)

2. stylesheet in a separate file (what we will mostly be using)

3. stylesheet inside the HTML (can be done for small projects but not ideal)

4. inline style in an element (can also be done but should be avoided)

Stage 3 Notes

Abstraction / Program With Objects

Object Oriented Programming
Leverages objects rather than actions, manipulates collections of objects, and is more data-oriented than logic-oriented.

Advantages to using Object Oriented Programming in large programs:
-- More modular than procedural programming;
-- Extensible, as objects can be modified to include new attributes & behaviors, and can be reused across multiple applications.
-- The reuse of modules saves development time and makes the code easier to maintain.
Introduction to Abstraction
The hiding of detail in programming is Abstraction.

Abstraction allows us to focus on the program we actually want to write and to use modules within the Python Standard Library without knowing the details of the programming in those modules.
Empowers us to use those functions by reading their documentation to learn about the functions.
Python Modules
A Module is a python file with some functions or variables in it.
We use "import" to import a module, then we can access the functions or variables in that module with the .(dot) operator.
Python Standard Library
A library usually refers to a collection of code made by others so it can be reused.
The Python standard library provides this functionality along with standardized solutions for problems that occur in programming. It has built-in modules that provide access to functionality otherwise inaccessible to Python programmers.
Use and Make Classes
A class is a logical grouping of data & functions (frequently referred to as "methods" when defined within a class).
A class is like a neatly packaged box that puts things together really well, a blueprint for creating objects.
A class is something that contains structure - it defines how something should be laid out or structured, but doesn't actually fill in the content.

An object is a particular instance of a class made up of variables, functions, and data structures.

An instance is a specific copy of the class that contains all the content, specifies what the content is.

Constructor is a special type of subroutine called to create an object and prepares the new object for use.

Self refers to the specific object being created from the class.

Class Attribute: a property of a class that is from composition and is usually a variable
There are five Predefined Class Attributes in Python:
-- Python Predefined Class Attributes
Of these five, we learned specifically about three in this lesson:
1. __doc__ is used to provide documentation for a class. Add documentation information directly at the beginning of the class.
2. __name__ is used to get the name of the class.
3. __module__ is used to get the name of the module in which this class was defined.

Class Variables is a variable defined in a class. They are accessed by all instances of that class.
If a value of a class variable is a constant (will not change), the name of the variable should be in ALL CAPS to define a variable like that ("VALID_RATINGS" vs "valid_ratings")

Instance variable is a variable defined in a class, of which each object of the class has a separate copy.

Class Variables vs. Instance Variables

Instance Method is a function defined inside a class & associated with an instance
Class vs Object vs Instance
Class is analogous of a blueprint
Object is the actual thing built from that blueprint
Instance is a virtual copy of the object
eg:
-- a class would be like a blueprint of a building
-- an object would be like any buildings built from that blueprint
-- an instance would be like a specific building usng that blueprint, such as apartment building or office building
